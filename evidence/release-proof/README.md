# Build Week competition proof

Competition proof records the exact commit and tested signed owner build. It
does not claim notarization, public release, clean-machine distribution, or
production readiness. Scenario count must be nonzero, every claimed scenario
must pass, and blockers must be empty. Product model steps record exact
`gpt-5.6-sol` with `high` reasoning. Private bodies, credentials, raw imports,
message contents, and user data are never committed.

## Required complete scenarios

- One connected story: confirmed Passport/ZIP Memory → proactive suggestion →
  clarify/propose → confirmed Mission → reviewed GitHub Skill → real
  iMessage/Discord participation → real Reminders write/readback → completion
  Evidence → Receipt → second similar success → Workflow Candidate.
- Real bidirectional iMessage and Discord with approved pairings, durable
  dedupe, restart recovery, and no duplicate send.
- One real official ChatGPT ZIP import, including category selection,
  minimized-excerpt approval, candidate review, one-hour raw cleanup, and a
  personalized Outcome. Competition V1 is OpenAI-only; Claude and cross-
  provider import are excluded and not claimed.
- Quick Passport and the ChatGPT ZIP source pass local secret scrubbing before any
  preview/provider request; captured provider bodies contain no raw, encoded,
  split, or boundary-obfuscated credential material.
- Source revocation atomically stops queued/local/model work, blocks future
  reads/suggestions, deletes raw/catalog/Candidate state, disables derived
  Memory, dismisses suggestions/proposals, pauses active dependent Missions,
  removes bookmark access, and invalidates dependent Workflow Candidates
  across restart while preserving only body-free Evidence/Receipt audit.
- One real public GitHub Skill import through audit, promotion, Mission use,
  update proposal, approval/rejection, and rollback; scripts remain rejected.
- Skill prompt-injection preflight and Mission/digest/manifest revalidation
  reject authority, secret, scope, tool, recipient, model, retention, and effect
  changes before model/effect work. Captured provider bodies keep Skill text in
  its digest-labelled untrusted-data field across restart and update.
- GitHub URL/redirect/streaming tests prove exact HTTPS host allowlisting, no
  cross-origin credential forwarding, immutable commit resolution, and byte
  limits before buffering.
- Agent Understanding eval with at least sixty varied human expressions and
  final Owner-approved communication rubric.
- Global Off proving no new listener, model, Workflow, or outbound effect can
  begin; sleep/offline/crash recovery must remain bounded and honest.
- Existing protected broker, tmpfs, administrator/cross-UID, and Developer-ID
  identity proofs remain required for the tested owner build. Notarization,
  staple, Gatekeeper public release, and clean-machine release proof are not
  competition gates and must not be claimed.

## Ineligible substitutes

Mocks, fixtures, screenshots, CI success, component-only probes, schema tests,
ad-hoc packages, and signature inspection are supporting evidence only. They
cannot satisfy a real provider or connected product scenario.
