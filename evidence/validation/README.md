# Build Week product validation

Only consent-safe aggregates belong here. Raw participant messages, receipts,
memory imports, names, emails, phone numbers, Slack/Discord IDs, and recordings
remain local and are never committed. Every published row requires the user's
explicit consent for that anonymous aggregate.

Formal three-user, 3/3 first completion, 2/3 48-hour reuse, and clean-install
validation are post-competition production work. They are not gates for
`BUILD_WEEK_COMPETITION_READY` and must not be claimed complete.

Competition validation records only consent-safe local observations for the
connected story: time to first useful suggestion, whether `Why this?` made the
source understandable, whether `Need you` asked one meaningful question,
whether the Receipt contained real Evidence, and whether the Workflow
Candidate followed two similar successes.

Passport and Deep ZIP validation records whether the ChatGPT source and OpenAI
destination, retention, candidate review, correction, Forget/disable-for-
suggestions, cancellation, external ZIP ownership, and raw deletion were
understood. Competition V1 is OpenAI-only; Claude and cross-provider import
are excluded and not claimed. It also records whether category selection and the
minimized-excerpt model boundary were understandable, and proves source
revocation stops work and disposes or disables every derived state. Channel validation covers only explicitly addressed
iMessage/Discord Mission traffic; ambient observation is excluded.

Revocation validation includes pending suggestion/proposal dismissal, bookmark
capability removal, active dependent Mission pause before effects, late-result
discard, restart, and preservation of only minimal body-free audit Evidence.

Validation also records local secret-scrubber results by non-body reason/count
only and verifies provider-body capture excludes raw, encoded, cross-entry,
and boundary-obfuscated credentials. No matched value enters screenshots,
logs, fixtures, committed evidence, or model-visible previews.

Skill validation records typed instruction-conflict reasons without retaining
malicious bodies, verifies the digest-labelled untrusted prompt boundary, and
checks that update/restart cannot substitute a different digest. GitHub
network validation records only host/result metadata for malicious URL,
redirect, streaming-limit, and credential-forwarding cases.

Agent Understanding validation includes at least sixty typo, transcription,
fragment, complaint, hesitation, joke, correction, age, and communication-
style cases. Failure is reported as failure and fixed within the approved
scope; results are never rounded or rewritten.
